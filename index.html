wily-cli
The NPM package, wily-cli, is a Node CLI creator tool that I have published to compete with the top Node CLI parsers and creators. I have used some of those tools for other development projects, but I was never completely satisfied with the usage. There were things that I found that could be handled automatically and not propagated to the user, and none of the the CLI packages provided a wide variety of CLI features, or were missing on opportunities. Users would need to use multiple CLI packages to acheive their goal. My package aims to be easier to use and setup, be feature rich, maintain acceptable and fast performance, provide many customization options, all while having zero dependencies: everything it does, it handles by itself, and it doesn't bloat a dependent's dependencies. Ideally, it should be the only CLI package you need to set up your CLI. Some features includes sub commands, on/off switch options, interactive CLI, execute JavaScript or binary command files, customizable strings with color effects, and more. Results from simple tests show that wily-cli has a similar run time with negligible difference when compared to the current top CLI tools, while requiring fewer lines of code to set up.

JsonsonsonLibgdx
JsonsonsonLibgdx is a LibGDX library I'm working on to simplify and quicken 2D cross-platform game development for Android, iOS, and desktop. The library supports management of static and animated images, applied animations, text/fonts, physics, particle systems, progress bars, ChartBoost advertisements, pixel perfect collisions, and game data (preferences and encrypted data). The library handles memory and texture management, as well as the ability to persist textures across screens without having to dispose them. Following the setup guide and extending the required classes, a game equipped with a powerful set of tools can be made ready to go for Android, Desktop, and iOS with little extra work for the developers.

Technologies

Java, LibGDX, Gradle

[m]pirik Health Tracker
Health Tracker is an Android healthcare application I developed with my Senior Design group at Milwaukee School of Engineering for [m]pirik. It's focused around bringing post-procedure health data collection into the comfort of the patient's home by tracking health data from multiple sources, such as Fitbit. It also allows the patient to perform exercises at home, allowing doctor and clinic visits to focus on the patient's progress and the next steps to recovery, as opposed to filling time with exercises. It contains the Android application, and a server, both of which I have worked on, but my primary work was done on the Android application.
